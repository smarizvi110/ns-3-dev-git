# Experiment Paper Artifacts (CATS vs Baseline TCP)

This directory contains the reproducible artifacts for the experiment comparison used in the paper.

## Scope

Included:

- Experiment source variants (CATS and Baseline TCP)
- Priority completion TSV outputs used for figures
- Plot generation script
- Generated comparison figures

## Directory Layout

- `experiments/`
  - `experiment-core-prioritization.cc` (CATS variant)
  - `experiment-baseline-tcp.cc` (Baseline TCP variant)
- `results/`
  - `cats/priority_completion.tsv`
  - `cats/parameter_debug.log`
  - `baseline/priority_completion.tsv`
  - `baseline/parameter_debug.log`
- `analysis/`
  - `plot_experiment_comparison.py`
- `plots/`
  - `completion_comparison.png`
  - `throughput_comparison.png`

## Important Implementation Note

CATS integration depends on modifications made directly inside ns-3 TCP internals, specifically:

- `src/internet/model/tcp-socket-base.cc`
- `src/internet/model/tcp-socket-base.h`

Those core changes are part of this repository state and are not duplicated in this folder.

## Running the Experiment Variants

Use the provided runner from repository root:

```bash
./paper/experiment/run_experiment_variants.sh
```

To run the same workflow with PCAP generation enabled:

```bash
ENABLE_PCAP=1 ./paper/experiment/run_experiment_variants.sh
```

What it does:

- Synchronizes tracked sources from `paper/experiment/experiments/` into `scratch/`
- Cleans, reconfigures, and rebuilds ns-3 using debug/assert/log/test settings
- Runs both variants
- Writes outputs into `paper/experiment/results/cats/` and `paper/experiment/results/baseline/`
- Optionally enables PCAP output when `ENABLE_PCAP=1`

Each run produces `priority_completion.tsv` in its target output folder.

### Run Artifacts Produced

For each variant output directory (`results/cats/` and `results/baseline/`):

- `priority_completion.tsv`: priority-group completion events used for plots.
- `parameter_debug.log`: run-time parameter/configuration trace (transport setup, selected options, and debug notes emitted by the scenario code).

### Understanding `parameter_debug.log`

These logs are intentionally kept as a lightweight audit trail for each run.

Why they are there:

- Record resolved run settings (network/link parameters, output path, and PCAP on/off).
- Provide a quick sanity check that CATS and baseline were launched with the expected options.
- Help explain outliers in `priority_completion.tsv` without immediately re-running full simulations.

What they typically show:

- Header metadata (`Experiment`, `Timestamp`, and full command-line arguments).
- Resolved run settings (`simTime`, link rates/delays, output directory, and `enablePcap`).
- Transport setup checkpoints from scenario code (BBR/PRR/RTT setup and CATS socket registration events).
- In CATS runs, a parameter reference block listing CATS tunables and their defaults.

How to use them:

- Compare `results/cats/parameter_debug.log` and `results/baseline/parameter_debug.log` first when results look unexpected.
- Confirm run settings match what you intended before trusting plots.
- Correlate log events with `priority_completion.tsv` and, when enabled, PCAP traces.

Scope note: the CATS parameter list in the log is a reference of defaults/available knobs unless you explicitly pass overrides in the run command.

Important: these are diagnostic traces, not the primary paper metric. The metric of record remains `priority_completion.tsv`.

### PCAP Files: Where They Go and Why You May Not See Them

The experiment sources support PCAP generation, but the current paper runner uses explicit `--outputDir=...` and does not pass `--enablePcap=true`.
Because of that, PCAPs are not generated by default in `paper/experiment/results/**`.

If you run with `--enablePcap=true`, PCAPs are written inside the chosen output directory:

- CATS variant prefix: `experiment-core-prio-*.pcap`
- Baseline variant prefix: `experiment-baseline-tcp-*.pcap`

Example manual runs from repository root:

```bash
./ns3 run "experiment-core-prioritization --outputDir=paper/experiment/results/cats --enablePcap=true"
./ns3 run "experiment-baseline-tcp --outputDir=paper/experiment/results/baseline --enablePcap=true"
```

Quick inspection examples:

```bash
ls paper/experiment/results/cats/*.pcap paper/experiment/results/baseline/*.pcap
tcpdump -tt -nn -r paper/experiment/results/cats/experiment-core-prio-0-0.pcap | head
tcpdump -tt -nn -r paper/experiment/results/cats/experiment-core-prio-0-0.pcap | grep "unknown-253" | head
```

Notes:

- Interface index and file numbering can vary (`-0-0`, `-1-0`, etc.).
- CATS priority tagging visibility in packet decode depends on dissector support and link/device context.

### Viewing CATS Priority in PCAPs (Wireshark)

When analyzing CATS PCAPs in Wireshark/ns-3 traces, add a custom column with field:

- `tcp.options.experimental[2]`

This exposes the CATS priority byte carried under the TCP experimental option header.

### Batch Extraction with tshark

If you want tabular output for analysis scripts, you can extract key fields with `tshark`.

Example (single PCAP):

```bash
tshark -r paper/experiment/results/cats/experiment-core-prio-0-0.pcap \
  -T fields \
  -e frame.number \
  -e frame.time_relative \
  -e tcp.stream \
  -e ip.src \
  -e ip.dst \
  -e tcp.seq \
  -e tcp.options.experimental[2] \
  -Y "tcp && tcp.options.experimental" \
  -E header=y -E separator=, -E quote=d
```

Example (all CATS PCAPs into one CSV):

```bash
for f in paper/experiment/results/cats/experiment-core-prio-*.pcap; do
  tshark -r "$f" \
    -T fields \
    -e frame.time_relative \
    -e tcp.stream \
    -e tcp.options.experimental[2] \
    -Y "tcp && tcp.options.experimental" \
    -E header=n -E separator=, -E quote=d
done > paper/experiment/results/cats/cats_priority_trace.csv
```

Note: if your Wireshark/tshark build labels this option differently, inspect available fields with:

```bash
tshark -G fields | grep -i experimental
```

## Regenerating Plots

From repository root:

```bash
python3 paper/experiment/analysis/plot_experiment_comparison.py \
  --baseline paper/experiment/results/baseline/priority_completion.tsv \
  --cats paper/experiment/results/cats/priority_completion.tsv \
  --outdir paper/experiment/plots
```

## Notes on Metrics

The experiment variants now emit only priority-group completion metrics in TSV format:

```txt
SimTime(s)    PageID    Priority    CompletionTime(ms)    Event
```

This is the metric input used by the paper plots.
